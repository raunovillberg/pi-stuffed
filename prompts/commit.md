Commit our current work.

For the title: 
- Be concise, but do not sacrifice clarity. It's more important for the commit to be easily searchable later than to force it to be arbitrarily short.
- Follow the style of the current repository. 
-- Are the commit titles usually prefixed by category ("feat:", "(docs)" etc.) or a ticket number ("TVDEV-3125:" etc)? If yes, do the same. If not, do not add them.

For the message:
- Be detailed. Focus on the "why", not just the "how". 
- If the session was exploratory, mark down paths pursued but not taken as well, it's all useful history.
- Do NOT include any security keys, PII or anything else you wouldn't want to entrust a malicious stranger with.

If the uncommited changes consist of work from different, unrelated sessions, then split them into multiple commits.
